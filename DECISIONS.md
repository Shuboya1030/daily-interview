## 2026-02-12 - MVP核心决策

### 🎯 产品定位更新

**Daily Interview** = 智能PM面试题聚合平台 + AI高频识别

**核心理念：** 不求大而全，但求精而准

**解决的核心问题：**
- 📚 信息过载：牛客、知乎等平台信息大而散
- ⏰ 时间紧迫：面试前临时抱佛脚，压力大
- 🎯 缺乏重点：不知道哪些是真正的高频考点

---

### ✅ 已确认的决策

| 决策点 | 选择 | 原因/备注 |
|--------|------|-----------|
| **产品核心创新** | AI高频识别 | 使用LLM进行语义相似度检测，自动识别高频题 |
| **视图设计** | 双视图（高频精选+全部题目） | 满足快速突击和深度准备两种场景 |
| **OKR范围** | 仅保留KR1-3 | MVP聚焦核心指标：时效性、可信度、覆盖度 |
| **时效性定义** | 信息源发布时间 | 更容易爬取和验证 |
| **默认时间范围** | 最近3个月 | 平衡信息量和时效性 |
| **爬虫频率** | 每天1次 | 成本和时效性的平衡 |
| **数据范围** | 每次爬取过去3个月 | 保持题库新鲜度 |
| **公司范围** | FAANG + 独角兽 | 覆盖最热门的面试目标公司 |
| **行业分类** | 动态调整 | 初始分类后根据数据迭代 |
| **历史题目** | 保留并在独立Tab展示 | 满足深度准备需求 |
| **时间筛选** | 保留功能 | 用户可自定义时间范围 |
| **趋势分析** | Phase 1不做 | MVP聚焦核心功能 |

## MVP功能范围

### ✅ Phase 1 包含（P0功能）
1. **智能信息源爬取**（每天更新）
2. **AI高频识别系统** ⭐ 核心功能
   - LLM语义相似度检测
   - 自动合并相似题目
   - 频率统计和排序
3. **双视图展示**
   - 🔥 高频精选视图（默认）
   - 📚 全部题目视图（独立Tab）
4. **多维度筛选**
   - 按公司/时间/题型筛选
   - 公司类型标记（FAANG/Unicorn/Big Tech）
5. **题目详情页**
   - 来源链接
   - 频率统计
   - 相关公司

### ❌ Phase 1 不包含
- ❌ 趋势分析
- ❌ 个性化Dashboard
- ❌ 用户登录/注册系统
- ❌ AI生成的Insights
- ❌ 用户贡献内容（UGC）
- ❌ 题目收藏/标记功能
- ❌ 4小时通关计划

### 🔄 Phase 2 计划
- 基础Insights生成
- 趋势对比分析
- 高频题相似度算法优化

### 🔮 Phase 3 展望
- 个性化推荐
- 用户贡献内容（UGC）
- 4小时通关计划功能

## 设计约束

1. **100% grounding原则**：所有展示的题目必须来自认证的信息源
2. **时效性优先**：80%+的题目必须在用户指定时间范围内发布
3. **性能要求**：页面加载时间 < 2秒（目标）
4. **可扩展性**：设计需支持未来添加更多公司和信息源
